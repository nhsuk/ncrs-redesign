About this prototype

This prototype explores the introduction of the Birth Notification Application (BNA) to the redesigned SCRa. The BNA allows users with specific RBAC permissions to notify the birth of a child.

Those users with the necessary RBAC permissions will be given relevant navigation options when confirming their role upon login.

Our first prototypes will explore the creation of a birth notification for a mother with an existing PDS record. The first stages of the login journey have been removed for brevity in this prototype.

Version 4 update

We've tested a few variations of our user flows and believe that they work well, though we will continue to test them.

The most problematic issue we have encountered is with our "View birth notifications" list page. This page in the current Summary Care Record application contains lots of information and isn't responsive, meaning that if we were to lift and shift the existing functionality we wouldn't satisfy the necessary accessibility requirements.

We also found that users of version 3 did not realise that the mother's name was a clickable link to view the birth notification.

As such, we have introduced a redesigned page in version 4 which we intend to test.

Version 4 hypotheses

If we present only the primary information that users require up front

Then they will still be able to identify a birth notification from the list

Because we are removing clutter from the page and giving the page a clear structure across all devices.


If we present secondary information that users require in an expandable field

Then users will be able to better identify the birth notifications they require

Because we are removing clutter from the page initially, whilst stil giving users the option to look for more information if they require it.


If we make the notification status a colour coded Action link

Then users will be able to identify at a glance any notifications which are in progress, failed or complete and use the link to view further information about each notification.

Because we know users react well to colour coded information, and we have seen proof that users identify Action links as interactive elements in previous prototypes.
